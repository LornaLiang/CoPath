# CoPath 前端界面设计说明

## 1. 前端定位

CoPath 前端是系统的用户交互层，负责展示学习目标、学习路径、AI 助手、知识图谱、学习画像和学习记录。

前端只负责展示和交互，不承担核心业务逻辑。

路径规划、学习画像更新、AI 状态分析、路径调整等逻辑必须由后端完成。

---

## 2. 技术栈

- React
- Vite
- Ant Design
- ECharts
- Axios
- React Router

---

## 3. 整体布局

系统采用后台管理式布局：

```text
左侧导航栏
+
顶部状态栏
+
主内容区域
```

### 左侧导航栏

包含以下菜单：

1. 学习目标
2. 学习路径
3. 我的学习
4. 知识图谱
5. 学习画像
6. 学习记录
7. 设置

左侧导航栏固定显示，当前页面高亮。

### 顶部状态栏

显示：

- 系统名称：CoPath
- 当前学生
- 当前学习目标
- 当前路径状态
- AI 助手状态

### 主内容区域

根据当前菜单切换不同页面。

整体风格要求：

- 现代
- 简洁
- 蓝白配色
- 卡片式布局
- 圆角
- 轻阴影
- 适合系统 Demo 展示

------

## 4. 页面一：学习目标页面

### 页面目标

帮助学生选择或确认学习目标。

### 页面内容

包含：

- 当前学生信息卡片
- 推荐学习目标列表
- 学习目标详情
- AI 学习目标建议
- 开始规划按钮

### 主要组件

1. 学生信息卡片

显示：

- 姓名
- 学习速度
- 学习偏好
- 当前学习状态

1. 学习目标卡片列表

每个目标显示：

- 目标名称
- 对应知识点
- 推荐等级
- 难度
- 简介

1. AI 建议区域

显示 AI 对学习目标的解释，例如：

> 根据你的学习记录，建议优先学习“递归”，因为你已经掌握函数基础，但调用栈和递归仍较薄弱。

1. 操作按钮

- 选择目标
- 开始路径规划

### 需要调用的接口

- GET `/api/students/current`
- GET `/api/goals`
- POST `/api/goals/select`

------

## 5. 页面二：学习路径页面

### 页面目标

展示系统为同一学习目标生成的多条学习路径，并支持查看当前路径和路径调整记录。

### 页面内容

包含：

- 当前学习目标
- 三条候选学习路径
- 当前推荐路径
- 路径节点进度
- AI 推荐理由
- 路径切换记录

### 主要组件

1. 路径概览卡片

显示：

- 当前目标
- 当前路径名称
- 当前学习节点
- 路径状态

1. 三条候选路径

至少展示三种路径：

- 基础补全路径
- 案例驱动路径
- 快速提升路径

每条路径以横向步骤条或图形方式展示。

当前推荐路径需要高亮。

1. 当前路径详情

展示：

- 路径节点列表
- 节点状态：未开始 / 学习中 / 已完成 / 补救节点
- 当前节点说明
- 推荐学习资源

1. AI 推荐理由

展示系统为什么选择当前路径。

1. 路径切换记录

展示：

- 原路径
- 新路径
- 触发原因
- 调整说明
- 时间

### 图表要求

可使用 ECharts 展示路径节点关系。

### 需要调用的接口

- GET `/api/paths/current`
- GET `/api/paths/candidates`
- GET `/api/paths/switch-logs`
- POST `/api/paths/switch`

------

## 6. 页面三：我的学习页面

### 页面目标

作为学生实际学习页面，支持当前知识点学习、AI 对话和学习反馈。

### 页面内容

包含：

- 当前学习节点
- 学习内容
- 学习资源
- AI 助手聊天窗口
- 学习反馈按钮
- 小测验区域

### 主要组件

1. 当前学习节点卡片

显示：

- 知识点名称
- 所属路径
- 难度
- 前置知识
- 当前学习进度

1. 学习内容区域

显示：

- 知识点说明
- 示例代码
- 讲解文本
- 推荐资源链接

1. AI 助手聊天区

支持：

- 学生输入问题
- AI 回复
- 展示历史对话
- 学习状态信号仅作为后台数据保存，演示界面不直接展示

1. 小测验区域

包含：

- 简单题目
- 提交答案
- 显示结果

1. 学习反馈按钮

包括：

- 我理解了
- 我还不理解
- 换个例子
- 降低难度
- 进入下一节点

这些反馈会触发后端更新学习画像或路径。

### 需要调用的接口

- GET `/api/learning/current`
- GET `/api/resources/current`
- GET `/api/dialogues`
- POST `/api/chat`
- POST `/api/learning/event`
- POST `/api/learning/feedback`

## 7. 页面四：知识图谱页面

### 页面目标

展示课程知识点及其依赖关系。

### 页面内容

包含：

- 知识图谱可视化
- 知识点详情
- 当前学习目标定位
- 当前路径高亮
- 前置知识展示

### 主要组件

1. 知识图谱图形区域

使用 ECharts Graph 展示节点与边。

节点表示知识点。

边表示前置依赖关系。

1. 节点详情面板

点击节点后显示：

- 知识点名称
- 描述
- 难度
- 所属章节
- 前置知识
- 后继知识
- 关联资源

1. 当前路径高亮

当前学习路径中的节点需要高亮显示。

当前正在学习的节点需要特殊标记。

### 需要调用的接口

- GET `/api/graph`
- GET `/api/graph/node/{node_id}`
- GET `/api/paths/current`

## 11. AI 助手交互设计

AI 助手主要出现在：

- 学习目标页面
- 学习路径页面
- 我的学习页面
- 学习记录页面

AI 助手不只是聊天窗口。

AI 输出应包含两类内容：

1. 自然语言回复
2. 结构化学习状态信号

前端演示界面只展示自然语言回复，保持“用户输入、AI 输出”的真实对话体验。

结构化信号仅由后端用于学习画像更新和路径规划，并保存到对话日志中，例如：

```json
{
  "knowledge_gap": "call_stack",
  "confusion_level": 0.8,
  "learning_preference": "example",
  "suggested_action": "insert_prerequisite"
}
```

## 12. 页面跳转流程

推荐流程：

```text
学习目标
  ↓
学习路径
  ↓
我的学习
  ↓
AI 对话与学习反馈
  ↓
动态调整路径
  ↓
学习画像更新
  ↓
学习记录沉淀
```

用户也可以通过左侧导航直接进入任意页面。

------

## 13. Demo 展示重点

前端必须能清楚展示以下系统特色：

1. 同一学习目标存在多条学习路径。
2. 不同学生有不同学习画像。
3. AI 能根据对话内容分析学习状态。
4. 系统能根据学习表现动态切换路径。
5. 路径调整有明确解释。
6. 学习画像会随学习过程变化。
7. 学习记录完整保存。

------

## 14. 设计约束

1. 前端不得硬编码核心业务逻辑。
2. 前端可以使用 mock 数据，但数据结构必须与后端 API 保持一致。
3. 所有页面必须支持从 API 加载数据。
4. 所有图表使用 ECharts。
5. 所有表格使用 Ant Design Table。
6. 所有按钮操作必须有明确反馈。
7. 所有页面必须适合 1366×768 屏幕展示。
8. 视觉风格应保持统一。

------

请 Codex 实现前端时严格遵循本文件。

1. 页面布局

3. 路由跳转
